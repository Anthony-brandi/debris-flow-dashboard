# 🌊 Debris Flow Dashboard

**A real-time post-fire debris flow risk assessment tool for emergency managers and geoscience researchers.**

Built for California wildfire response — integrating Google Earth Engine satellite analysis, empirical volume modeling, and live geospatial visualization into a single operational dashboard.

> **Deployed on:** Streamlit Community Cloud  
> **Intended Audience:** CAL FIRE, USGS, academic researchers

## 🔬 Scientific Background

This tool applies the **Gartner et al. (2008)** empirical model to estimate post-fire debris flow volume:

$$\ln(V) = 4.22 + 0.39\sqrt{i_{15}} + 0.36\ln(A) + 0.13\sqrt{B \cdot R}$$

| Variable | Definition | Source |
|---|---|---|
| $V$ | Predicted debris flow volume (m³) | Model output |
| $i_{15}$ | Peak 15-min rainfall intensity (mm/h) | Rain gauge / NWS |
| $A$ | Watershed area (km²) | USGS SRTM DEM |
| $B$ | Fraction of basin burned at moderate–high severity | Sentinel-2 dNBR |
| $R$ | Local relief — max minus min elevation (m) | USGS SRTM DEM |

**Reference:** Gartner, J.E., Cannon, S.H., Santi, P.M., & Dewolfe, V.G. (2008). Empirical models to predict the volumes of debris flows generated by recently burned basins in the western U.S. *Geomorphology*, 96(3–4), 121–139.

## ⚙️ Installation & Local Setup

### Prerequisites

- Python 3.10+
- A Google Earth Engine account with API access
- A GEE Service Account with a downloaded JSON key

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/Anthony-brandi/debris-flow-dashboard.git
cd debris-flow-dashboard

# 2. Install dependencies
pip install -r requirements.txt

# 3. Configure GEE credentials
mkdir -p .streamlit
cat > .streamlit/secrets.toml << EOF
GEE_SERVICE_ACCOUNT = "your-service-account@project.iam.gserviceaccount.com"
GEE_PRIVATE_KEY = "-----BEGIN RSA PRIVATE KEY-----\n..."
EOF

# 4. Launch the app
streamlit run app.py
```

> ⚠️ **Never commit your `.streamlit/secrets.toml` or any `.json` key files.**  
> The `.gitignore` in this repo is configured to exclude them.

## ☁️ Streamlit Community Cloud Deployment

1. Fork or push this repo to your GitHub account
2. Go to [share.streamlit.io](https://share.streamlit.io) and connect your repo
3. Set the main file path to `app.py`
4. Under **Advanced Settings → Secrets**, paste the contents of your `secrets.toml`
5. Click **Deploy**

## 📊 App Modules

| Module | Description |
|---|---|
| **1. Incident Briefing** | Fire metadata, perimeter map, mission parameters |
